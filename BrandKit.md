# BYKI Brand Kit

## Typography

### Primary Font: Satoshi

**Satoshi** is the primary typeface for the BYKI brand across all platforms.

#### Font Weights
- **Satoshi Regular** - Body text, secondary content
- **Satoshi Bold** - Headlines, primary content, emphasis

#### Character Set
```
ABCDEFGHIJKLMNOPQRSTUVWXYZ
abcdefghijklmnopqrstuvwxyz
```

## Color Palette

### Primary Colors

| Color | Hex Code | Usage |
|-------|----------|-------|
| **Dark Green** | `#004e3c` | Headers, navigation bars |
| **Primary Green** | `#28b55f` | Buttons, primary actions, highlights, icons |
| **Light Green** | `#28b660` | Gradient endpoint, accents |

### Neutral Colors

| Color | Hex Code | Usage |
|-------|----------|-------|
| **Black** | `#212123` | Headlines, primary body text |
| **Dark Gray** | `#7c7c7c` | Subheadlines, secondary body text, labels |
| **Medium Gray** | `#e0e0e0` | Backgrounds, secondary buttons, dividers |
| **Light Gray** | `#f6f6f6` | Canvas background, page backgrounds |
| **White** | `#ffffff` | Containers, cards, button text |

### Gradient System

**Primary Gradient** - Used for selected containers, featured elements, and emphasis
```
Linear Gradient: #004d3d → #28b660
Direction: Left to right or Top to bottom (context dependent)
```

## UI Components

### Buttons

#### Primary Button
- **Background**: `#28b55f` (solid) or gradient `#004d3d → #28b660`
- **Text**: `#ffffff` (white)
- **Font**: Satoshi Bold
- **Shape**: Rounded corners (pill shape)
- **Text Transform**: Sentence case or Title Case

#### Secondary Button
- **Background**: `#e0e0e0` (light gray)
- **Text**: `#7c7c7c` (dark gray)
- **Font**: Satoshi Regular
- **Shape**: Rounded corners

### Cards & Containers

#### Standard Card
- **Background**: `#ffffff` (white)
- **Border**: Subtle 1px or none
- **Border Radius**: 12-16px (rounded corners)
- **Shadow**: 3% drop shadow for depth
- **Padding**: 16-24px

#### Canvas Background
- **Background**: `#f6f6f6`
- Use for page backgrounds behind white cards

### Status Badges

- **Background**: `#28b55f` (primary green)
- **Text**: `#ffffff` (white)
- **Font**: Satoshi Bold
- **Style**: Small, rounded pill
- **Text Transform**: UPPERCASE or Title Case

## Brand Graphics

- **Style**: Layered upward-pointing chevrons
- **Colors**: Green gradient (light to dark)
- **Usage**: Logo element, loading indicators, branding accents
- **Format**: PNG with transparency

### Vehicle Imagery
- **Style**: Clean, modern car renders on subtle circular shadow/platform
- **Background**: White or light gray
- **Placement**: Centered within cards

---

## Layout Principles

### Spacing
- Use consistent padding/margin: 8px, 12px, 16px, 24px, 32px
- Maintain generous white space around key elements
- Cards should have breathing room from page edges

### Information Hierarchy
1. **Headlines**: Satoshi Bold, `#212123`, large size
2. **Subheadlines**: Satoshi Regular, `#7c7c7c`, medium size
3. **Body Text**: Satoshi Regular, `#7c7c7c`, standard size
4. **Labels**: Satoshi Regular, `#7c7c7c`, small size, sometimes uppercase

### Data Presentation
- Use clean label-value pairs
- Align related data in columns (2-3 column layouts)
- Monospaced presentation for technical data (VINs, codes)
- Clear visual separation between data groups

## Iconography

### Style
- **Primary Color**: `#28b55f` (green)
- **Style**: Clean, modern, minimal line icons or solid fills
- **Examples**: Checkmarks, gear/settings, chevrons

### Usage
- Use sparingly - icons should support, not overwhelm
- Maintain consistent stroke weight
- Pair with labels for clarity

---

## Visual Effects

### Shadows
- **Drop Shadow**: 3% opacity on white containers
- **Purpose**: Create depth and card elevation
- **Application**: Use subtly - avoid heavy shadows

### Progress Indicators
- **Bar Color**: `#28b55f` (green)
- **Background**: `#e0e0e0` (light gray)
- **Style**: Horizontal bars with rounded ends

---

## Voice & Tone

### Guidelines
- **Clear & Direct**: Avoid jargon, use plain language
- **Technical but Accessible**: Balance technical accuracy with user-friendliness
- **Helpful**: Guide users through diagnostics with confidence
- **Professional**: Maintain credibility in automotive diagnostics

### Example Phrases
- "Identify Your Vehicle"
- "Vehicle Diagnostics"
- "Continue to Scan"
- "Start Scan"
- "OBD-II scan live telemetry, fault analysis"

## Don'ts

❌ Don't use colors outside the defined palette
❌ Don't add decorative emojis or casual icons
❌ Don't use fonts other than Satoshi
❌ Don't use heavy drop shadows or effects
❌ Don't overcrowd interfaces - maintain white space
❌ Don't mix rounded and sharp corners inconsistently
